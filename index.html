Market Context & Problem Statement

Why this matters right now

The global talent market is experiencing its most severe skills mismatch in modern history. Students graduate with academic credentials that no longer map to what employers actually need ‚Äî and by the time they realize the gap, years have already been lost to outdated curricula and misaligned effort.

AI / ML Pipeline

How the intelligence layer works end-to-end

üì•
Stage 1: Data Ingestion Pipeline (Scrapy + Playwright)
Async web crawlers run daily across LinkedIn, Indeed, Naukri, Glassdoor, AngelList, and Wellfound. Each posting is parsed with custom NLP to extract: job title, required skills (hard + soft), experience level, salary range, company name, sector. Raw data hits Elasticsearch for full-text search, then skill occurrence frequencies are computed and stored in PostgreSQL as weekly snapshots. 15,000+ job postings processed per daily run. Anti-scraping handled via rotating residential proxies (Bright Data) and headless browser fingerprint randomization.
üîç
Stage 2: Skill Extraction (BERT + spaCy NLP)
Resume and job description text is passed through a fine-tuned BERT NER model (trained on 50K labeled resumes) to identify skill entities. Skills are then mapped to a canonical ontology of 1,200+ normalized skill nodes (e.g., "React.js", "ReactJS", and "React" all map to "react") using a custom knowledge graph. Each extracted skill is assigned a confidence score via cosine similarity of contextual embeddings (sentence-transformers/all-mpnet-base-v2). GitHub repos are analyzed for language usage, library imports, and README keywords to infer practical proficiency beyond self-reported skills.
üìê
Stage 3: Gap Analysis (Cosine Similarity + Graph Distance)
The student's normalized skill vector is compared against the aggregate skill requirement vector for the target role using cosine similarity in a 768-dimensional embedding space. Additionally, a prerequisite skill graph (built with NetworkX) computes graph distance between current skills and target skills to recommend optimal learning sequencing. Each gap is weighted by: (market demand index √ó 0.4) + (criticality √ó 0.35) + (salary premium √ó 0.25). Result is a ranked list of gaps with actionable priority scores, estimated learning hours (from historical completion data), and time-to-close projections.
üó∫Ô∏è
Stage 4: Roadmap Generation (GPT-4o + LangChain)
LangChain orchestrates a multi-step GPT-4o prompt chain: (1) Receive skill gap report + user constraints. (2) Query vector DB (Pinecone) for best learning resources per skill. (3) Apply topological sort on prerequisite graph for optimal sequencing. (4) Generate week-by-week plan with resources, hands-on projects, and verifiable milestones. (5) Self-critique loop: LLM validates the roadmap against time constraints and adjusts if overloaded. Final roadmap is stored in PostgreSQL and rendered as an interactive React Flow graph in the UI. Roadmaps auto-refresh monthly as market trends shift.
üîÆ
Stage 5: Future Forecasting (LSTM + XGBoost Ensemble)
Historical skill demand time-series (weekly snapshots from last 2 years) is fed into a multi-variate LSTM model to forecast demand trajectories for each skill. XGBoost adds feature-engineered signals: GitHub star growth velocity, Stack Overflow question volume, arXiv paper citations, and tech subreddit mention frequency. Ensemble predictions (LSTM 60%, XGBoost 40%) produce 6, 12, 24-month demand forecasts with confidence intervals. Skills predicted to surge in the next 6 months are flagged in the roadmap with a "Future-Proof" badge, giving learners a competitive head start on emerging technologies.
